How certificates work here

  1. Vault — this page holds one TLS identity (fullchain + key), encrypted. Never shown again.
  2. Service maps — each consumer (NPM volume, UniFi, mail, …) is a map: host + directory + filenames + owner/mode + optional restart.
  3. Deploy — {{ ph_brand() }} SSHs to the host, writes those exact files, applies permissions, runs your post-deploy command (e.g. container restart).

Example: map “NPM on edge” → /opt/stacks/npm/data/custom_ssl, layout pair, files fullchain.pem + privkey.pem, owner root:root mode 600, post-deploy docker compose -f /opt/stacks/npm/docker-compose.yml restart. Point the app at those paths (or the volume that mounts them).
